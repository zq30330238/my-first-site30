# 截图识别技能

读取并分析用户截图，提取关键信息，给出可执行的解决方案。

## 触发条件

- 用户发送截图（对话中出现 `[Unsupported Image]` 或图片路径）
- 用户说"你看"、"看看这个"、"截图"、"图片"、"帮我看下"
- 用户贴了错误日志/控制台输出的截图
- 用户贴了 AdSense/Google 后台截图

## 识别流程

### 1. 定位截图文件
Windows 截图通常在这些位置：
- `C:\Users\Administrator\AppData\Local\Temp\image_*.png`
- 用户手动粘贴的截图（对话中会有路径提示）
- 微信/QQ 截图保存在文档目录

### 2. 读取截图
使用 Read 工具读取图片文件：
```
Read(file_path="C:\Users\Administrator\AppData\Local\Temp\image_xxxxx.png")
```

### 3. 分析内容类型

| 截图类型 | 提取重点 | 输出 |
|----------|---------|------|
| 代码/终端错误 | 错误类型、文件名、行号 | 修复方案 |
| 网页/UI 界面 | 布局问题、文字内容、状态 | 设计建议 |
| 配置文件 | 参数值、格式 | 配置修正 |
| AdSense 后台 | 状态文字、按钮、提示 | 下一步操作 |
| 数据分析 | 数字、趋势 | 解读和建议 |
| 浏览器报错 | HTTP 状态码、错误描述 | 排查步骤 |

### 4. 如果读取失败
1. 尝试找最近修改的 temp 图片文件
2. 列出可能的文件路径让用户确认
3. 让用户手动描述截图内容

## 截图文件搜索策略

如果不知道确切路径，用 Bash 查找最近截图：
```bash
ls -lt "C:/Users/Administrator/AppData/Local/Temp/image_*.png" | head -5
```

或者：
```bash
dir "C:\Users\Administrator\AppData\Local\Temp\image_*.png" /O-D /B
```

## 回复格式

```
【截图分析】
- 类型: (代码错误/UI问题/配置/数据)
- 关键内容: (提取的核心信息)
- 问题: (如果需要解决什么问题)
- 方案: (具体操作步骤)
```

## 注意事项
- 截图中的敏感信息（密码、token）不要显示在回复中
- 如果截图不清，列出多种可能的判断
- 优先给出可执行的操作，不是泛泛建议
- 涉及修改代码时，准备好直接动手帮改
