# 综合增强技能

在复杂场景下自动启用高效工作模式，最大化发挥 Claude Code 能力。

## 触发条件

以下场景自动激活：
- 项目文件数 > 50
- 单次任务涉及 5+ 文件修改
- 批量搜索替换
- 跨目录操作
- 用户说"全力"、"火力全开"、"用最强模式"
- 长时间复杂任务

## 增强能力清单

### 1. 全局感知增强
- 操作前自动 Glob/Grep 了解影响范围
- 修改文件前读取确认当前状态
- 自动识别相关依赖文件

### 2. 并行执行增强
- 独立任务自动拆分为并行 Agent
- 3+ 个独立目录操作 → 每个目录一个 Agent
- 批量文件修改 → 按目录分组并行

### 3. 批量操作增强
- 同类修改合并为一次批量编辑
- 使用 replace_all 处理全局替换
- 避免逐个文件手动修改

### 4. 质量保障增强
- 改动完成后自动抽样验证
- 检查所有受影响文件的引用完整性
- 自动运行相关测试

### 5. 上下文管理增强
- 长任务自动使用 TodoWrite 追踪进度
- 阶段性向用户汇报关键进展
- 任务中断后自动记录断点

## 工作流程

```
接收任务
  ↓
全局搜索 → 了解影响范围
  ↓
制定执行计划 → TodoWrite 记录
  ↓
拆分任务 → 并行 Agent（如适用）
  ↓
逐个执行 → 实时更新进度
  ↓
全局验证 → 抽样检查
  ↓
汇报结果
```

## 关键原则

1. **先搜后改**：永远先了解全貌再动手
2. **能并行不串行**：独立任务坚决并行
3. **改完必验**：不假设改动一定正确
4. **进度可见**：复杂任务让用户知道在干嘛
5. **断点可续**：万一中断能接上

## 适用场景
- 网站矩阵批量建站
- 跨项目全局重构
- 大规模文件迁移
- 复杂 SEO 优化
- 任何"一个人搞不定"需要 AI 全速运转的任务
