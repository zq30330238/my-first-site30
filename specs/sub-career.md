## CTR与流量增长策略

### 标题优化 (Title CTR)
- 标题含数字: "7 Proven Strategies to..." / "5 Resume Mistakes Costing You..."
- 含强力词: Ultimate, Essential, Complete, Proven, Expert-Backed
- 含年份: "The 2026 Guide to..." (年初更新年份)
- 问题式标题: "How Much Should You Ask for in a Salary Negotiation?"
- 长度≤60字符(Google显示上限)，核心关键词前移

### Meta Description — 搜索引擎中的"广告文案"
- 150-160字符，每页唯一
- 含CTA动词: Learn how / Discover / Master / Find out / Get
- 含主关键词(粗体显示提高视觉突出)
- 明确告诉用户点进来能得到什么价值
- 例: "Learn 7 proven salary negotiation scripts that helped professionals earn 20% more. Master counter-offer strategies and market rate research. Read our complete guide."

### Featured Snippet 目标
- **段落摘要**: 每篇文章在Key Takeaways框内用40-60字精准回答核心问题
- **列表摘要**: FAQ用 `<ol>` / `<ul>` 编号列表，每项简洁(1-2句)
- **表格摘要**: 对比数据用语义化 `<table>`，表头清晰
- Q&A格式: h2问句 → 紧跟p回答，Google偏好这种结构

### 内链策略 (Session Depth)
- 每篇文章自然嵌入2-3个上下文链接(锚文本含目标关键词)
- 文章底部 "Related Articles" (3篇，动态读取同分类)
- 文章末尾 "Read Next" 引导下一篇(降低跳出率)
- 首页→分类→文章→相关文章→首页，形成闭环
- **孤岛页面零容忍**: 每页至少被2个其他页面链接

### 内容新鲜度 (Freshness Signal)
- 每篇文章显示 "Last updated: YYYY-MM-DD"
- 每年Q1批量更新 "2026" → "2027" 年份引用
- 定期轮换首页Featured/Latest文章(每周)
- 内容站: 保持每日更新节奏(谷歌偏好活跃站点)

### 回访机制 (Return Visitor)
- 内容系列: 同一分类文章标注 "Part of our Resume Mastery Series"
- 首页 "Trending Now" 区块推荐热门内容
- 打印友好样式(用户保存=品牌持续曝光)

### 富文本摘要 (Rich Snippets)
- FAQ Schema: 每篇含3-5个Q&A → 搜索结果显示可展开FAQ
- HowTo Schema: 步骤类文章(简历写作/面试准备)加HowTo结构化数据
- BreadcrumbList Schema → 搜索结果显示面包屑路径
- 搜索结果星级: 暂无(需要真实评价数据)，但Review Schema预留

### 移动端CTR
- 移动端标题比桌面端显示更短(≤55字符实际)
- 移动端meta description显示约120字符，关键信息前置
- 移动端加载速度<2秒(纯静态HTML天然达标)
- 触摸目标≥44x44px(按钮/链接)

## 质量与UX标准

### 内容质量
- 每篇文章 ≥1200字，含实质性建议/步骤/模板
- 文章结构: 导语钩子 → ToC目录 → 核心要点框(Key Takeaways) → 正文3段 → FAQ → 结论
- 阅读时间标注: 每篇文章顶部显示 "X min read" (按200词/分钟计算)
- 最后更新日期: 文章底部显示 "Last updated: YYYY-MM-DD"
- 禁止AI cliché: "In this guide, we'll" / "Let's dive in" / "Moreover" / "In conclusion" / "whether you're a" / "delve into" / "in today's fast-paced world"
- 引用真实数据源(Glassdoor/LinkedIn/BLS)，不确定的注明"general career advice"

### 视觉质量
- 封面图统一16:9 (1200x675)，所有图片 lazy loading (`loading="lazy"`)
- 图片alt文本描述图片内容(SEO+无障碍)，不得为空或仅写"image"
- 色彩对比度 WCAG AA: 蓝色`#2563EB`在白色背景上对比度5.3:1(通过)
- 图片上禁止任何阴影遮罩(bg-black/, gradient-overlay, rgba叠层)
- 简历模板表格: 斑马条纹(zebra-stripe)，移动端横向滚动

### UX设计
- **面包屑导航**: Home > Category > Article Title (每篇文章顶部)
- **文章内目录(ToC)**: 桌面端sticky侧边栏，h2自动生成锚点链接，移动端折叠
- **相关文章**: 每篇文章底部3篇推荐，动态读取同分类真实文章标题
- **返回顶部按钮**: 纯CSS实现(锚点#top)，右下角固定
- **阅读进度条**: 页面顶部2px高，品牌色，纯CSS sticky
- **移动端汉堡菜单**: 纯CSS checkbox实现，无JS
- 外部链接 `target="_blank" rel="noopener noreferrer"`
- 所有链接hover状态有视觉反馈(颜色过渡+下划线)

### SEO
- **Schema.org**: Article + BreadcrumbList + Organization + WebSite + FAQ(每篇文章)
- **Meta description**: 150-160字符，含主关键词 + CTA
- **OG标签**: og:title / og:description / og:image(1200x630) / og:url / og:type=article
- **Twitter Card**: `summary_large_image`
- **Canonical URL**: 每页自引用
- **标题层级**: h1(文章标题1个) → h2(段落标题) → h3(子段)，绝不跳级
- **内部链接**: 每篇文章至少链接2-3篇同站其他文章(锚文本含关键词)
- Sitemap.xml 含所有页面，优先级: index=1.0, article=0.8, static=0.5
- robots.txt 不block任何资源

### 广告位策略
- 位置: 导语后(After Intro) + 正文中(Between sections) + 结论前(Before Conclusion) = 最多3个/篇
- 纯Google自动广告(`<script async src="https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-XXXX">`)，含 `crossorigin="anonymous"`
- 广告不打断阅读流，不在首屏(viewpoint)堆叠
- 首页: hero下方 + 文章网格中间 + footer上方 = 3个
- 禁止手动广告位(`<ins class="adsbygoogle">` + `data-ad-slot`)

### 性能
- Tailwind CSS CDN (已gzip压缩)
- 预连接DNS: `<link rel="preconnect" href="https://cdn.tailwindcss.com">` + `<link rel="dns-prefetch" href="https://pagead2.googlesyndication.com">`
- 图片 loading="lazy" (首屏3张除外)
- HTML最小化: 无多余空格/换行/注释/空div
- 零JS框架，零依赖，纯静态HTML

### 无障碍(A11y)
- 语义化HTML: `<header>` / `<nav>` / `<main>` / `<article>` / `<section>` / `<aside>` / `<footer>`
- **Skip to content**: `<a href="#main-content" class="sr-only focus:not-sr-only">Skip to content</a>`
- 所有`<img>`有`alt`属性(装饰图alt=""，内容图描述性文本)
- 表单(input/textarea)有`<label>`关联，按钮有描述性文本
- 键盘可导航: Tab顺序合理，hover/focus状态有`:focus-visible`样式
- 颜色不是唯一信息传递方式(错误提示同时用图标+文字+颜色)
